# INT176 Export Error: Session Initialization Error

If you see the error message:

**“INT176 Export Error: Session initialization error. Please reach out to Concierge for additional support.”**

It means there is a session or connection issue between Expensify and Sage Intacct that is preventing the export from starting.

---

## Why the INT176 Export Error Happens

The INT176 export error indicates a communication or authentication issue between Expensify and Sage Intacct.

This can occur if:

- The integration session cannot be initialized  
- Authentication credentials are temporarily invalid  
- The connection between systems is interrupted  

Because this error relates to the integration session itself, it cannot be resolved by updating report data.

---

# How to Fix the INT176 Export Error

This error requires additional investigation.

Please reach out to **Concierge** for support. Include:

- The full error message  
- The report name or ID  
- The time the export was attempted  

Concierge will review the integration logs and help restore the connection so you can successfully export your report.

---

# FAQ

## Can I fix this by running Sync?

No. This error is related to session initialization and typically requires backend review.

## Does this mean the integration is disconnected?

Not necessarily. It indicates a session issue, but Concierge can confirm whether the connection needs to be refreshed or reconfigured.
