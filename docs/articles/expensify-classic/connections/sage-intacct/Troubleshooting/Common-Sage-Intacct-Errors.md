# How to troubleshoot common Sage Intacct errors

When your Sage Intacct integration encounters an issue, Expensify displays an error code in your report comments or in a notification to the Preferred Exporter. Use this page to find your error code and follow the link to the detailed troubleshooting steps.

---

## How to resolve Sage Intacct export errors

- **INT009**: Update manager approval settings in Sage Intacct before retrying the export. [Learn how to fix the INT009 error](Export-Errors/INT009-Export-Error.md).
- **INT012**: Update Sage Intacct settings when the Reason for Expense field is required. [Learn how to fix the INT012 error](Export-Errors/INT012-Export-Error.md).
- **INT028**: Add a required Location to the employee profile before retrying the export. [Learn how to fix the INT028 error](Export-Errors/INT028-Export-Error.md).
- **INT035**: Change the export date or update permissions when the accounting period is closed. [Learn how to fix the INT035 error](Export-Errors/INT035-Export-Error.md).
- **INT043**: Update xml_gateway user or role permissions in Sage Intacct. [Learn how to fix the INT043 error](Export-Errors/INT043-Export-Error.md).
- **INT054**: Ensure the employee record and email match before retrying the export. [Learn how to fix the INT054 error](Export-Errors/INT054-Export-Error.md).
- **INT073**: Configure required account fields or tags before retrying the export. [Learn how to fix the INT073 error](Export-Errors/INT073-Export-Error.md).
- **INT083**: Select or map categories that support billable expenses in Sage Intacct. [Learn how to fix the INT083 error](Export-Errors/INT083-Export-Error.md).
- **INT084**: Verify that all tags and dimensions are active in Sage Intacct before retrying the export. [Learn how to fix the INT084 error](Export-Errors/INT084-Export-Error.md).
- **INT088**: Reopen the Cash Management subledger in Sage Intacct before retrying the export. [Learn how to fix the INT088 error](Export-Errors/INT088-Export-Error.md).
- **INT149**: Resolve missing or duplicate employee records in Sage Intacct before retrying the export. [Learn how to fix the INT149 error](Export-Errors/INT149-Export-Error.md).
- **INT176**: Resolve when a session or connection issue prevents exporting to Sage Intacct. [Learn how to fix the INT176 error](Export-Errors/INT176-Export-Error.md).
- **INT194**: Enable and map billable expenses in the Workspace before retrying the export. [Learn how to fix the INT194 error](Export-Errors/INT194-Export-Error.md).
- **INT205**: Ensure projects are properly linked to customers in Sage Intacct before retrying the export. [Learn how to fix the INT205 error](Export-Errors/INT205-Export-Error.md).
- **INT225**: Properly import and configure tax rates in the Workspace before retrying the export. [Learn how to fix the INT225 error](Export-Errors/INT225-Export-Error.md).
- **INT304**: Correct invalid categories or tags on $0.00 expenses before retrying the export. [Learn how to fix the INT304 error](Export-Errors/INT304-Export-Error.md).
- **INT446**: Remove the On Hold status from a vendor in Sage Intacct before retrying the export. [Learn how to fix the INT446 error](Export-Errors/INT446-Export-Error.md).
- **INT459**: Resolve missing or inactive categories in Sage Intacct before retrying the export. [Learn how to fix the INT459 error](Export-Errors/INT459-Export-Error.md).
- **INT480**: Associate a report submitter with the correct project in Sage Intacct before retrying the export. [Learn how to fix the INT480 error](Export-Errors/INT480-Export-Error.md).
- **INT503**: Enable Expense Type in Sage Intacct before retrying the export. [Learn how to fix the INT503 error](Export-Errors/INT503-Export-Error.md).
- **INT592**: Configure Smart Rules at the entity level in Sage Intacct before retrying the export. [Learn how to fix the INT592 error](Export-Errors/INT592-Export-Error.md).
- **INT664**: Ensure the vendor record email exactly matches the report submitter's email before retrying the export. [Learn how to fix the INT664 error](Export-Errors/INT664-Export-Error.md).
- **INT680**: Categorize uncategorized expenses before retrying the export to Sage Intacct. [Learn how to fix the INT680 error](Export-Errors/INT680-Export-Error.md).
- **INT771**: Update export date settings or permissions when the reporting period is closed. [Learn how to fix the INT771 error](Export-Errors/INT771-Export-Error.md).
- **INT860**: Remove restrictions from a GL account linked to a credit card before retrying the export. [Learn how to fix the INT860 error](Export-Errors/INT860-Export-Error.md).
- **INT993**: Resolve when a connection issue prevents creation of a cctransaction record. [Learn how to fix the INT993 error](Export-Errors/INT993-Export-Error.md).

# FAQ

## Where do I find my Sage Intacct error code?

When an export or sync fails, the error code appears in the report comments and in an email notification sent to the Preferred Exporter configured in your Workspace's **Accounting** settings.

## What should I do if my error code is not listed here?

If your error code does not appear on this page, try syncing your Workspace with Sage Intacct first. If the issue persists, contact Concierge for assistance and include the full error message.
