# INT176 Export Error in Sage Intacct Integration

If you see the error:

INT176 Export Error: Session initialization error. Please reach out to Concierge for additional support.

This means there is a connection issue between Expensify and Sage Intacct that is preventing the export from starting.

---

## Why the INT176 Export Error Happens in Sage Intacct

The INT176 error indicates a session or authentication issue between Expensify and Sage Intacct.

This can occur if:

- The Sage Intacct session cannot be initialized.
- The integration connection is temporarily interrupted.
- There is a configuration or authentication issue with the integration.

Because this error relates to the integration session itself, it cannot be resolved by updating report data.

This is a session or connection issue, not a tag validation or employee record error.

---

## How to Fix the INT176 Export Error

This error requires additional investigation.

Reach out to **Concierge** for support and include:

- The full error message.
- The report name or report ID.
- The time the export was attempted.

Concierge will review the integration logs and help restore the connection so you can successfully export your report.

---

# FAQ

## Can I Fix This by Running Sync Now?

No. This error is related to session initialization and typically requires additional investigation.

## Does This Mean My Sage Intacct Integration Is Disconnected?

Not necessarily. It indicates a session issue. Concierge will confirm whether the connection needs to be refreshed or reconfigured.
